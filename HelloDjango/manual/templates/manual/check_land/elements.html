Реквизиты и Полиси

В низу страницы могуть быть реквизиты и ссылки на Потилику конфиденциальности и Пользовательское соглашение

{% img %} https://i.imgur.com/G36QMdj.png {% end %}

На лэнде реквизиты должны быть обязательно, на прелендингах их быть не должно

{% notice %} Если лэнд выглядит как прелэнд, реквизиты не нужны. {% end %}

Полиси должны быть рабочими - при клике открываеться окошко с текстом. Окно должно скроллиться, закрываться при нажатии на крестик или все окошка с тектом, быть адаптивным.

{% slider dark %} https://i.imgur.com/BnFnwXu.png https://i.imgur.com/V0A3RCL.png {%end%} {% note %} Полиси на прелэндингах На прелэндингах или на лэндах выглядящих как прелэнды, вместо открытия окна с полиси, на ссылках допускаеться скролл к форме или редирект (если это прелэнд). {% end %}
Локализация

Язык полиси должен совпадать с языком сайта. Если это Арабский язык(текст с права-налево) полиси также должны быть спара-налево.

Даты

Все даты на должны проставляться скриптами, те менять свое значение в зависимости от текущей даты

Даты по тексту должны быть в следующем формате:
  • 01.02.2021
  • 01.02.21
  • 2021.02.20
Некорректный формат:
  • 01-02-2021
  • 01/02/21
  • 2021\02\20
{%notice%} Допускаються дыты без указания года, либо с указанием названия месяца. {%end%}
Типы дат
Дата статьи, публикации

Дата публикации обычно находиться вначале сайта либо в шапке. Она может быть как сегодняшней, так и минус пару дней от текущей

Дата действия акции, предложения

Дата начала действия акции обычно начинается за неделю от текущей даны, а заканчиваеться сегодня, иногда с припиской (ВКЛЮЧИТЕЛЬНО)

Дата коментария

Дата оставления коментария должны быть не раньше даты публикации, и не позже сегодня

Вместо даты также может указываться сколько прошло времени после оставления коментария, например 2 часа назад

{% notice %} Допускаються коментарии без дат, но тогда без дат должны быть все коментарии. {%end%}
Даты по тексту

Значение даты в тексте должно соответсвовать контексту. Например: если реч идет об открытии иновационого метода для лечения чего либо, год открытия должен быть -1 -2 от текущего.

Также по тексту могут быть старые года

Скролл сайта

По вертикали сайт должен без проблем скролиться как колесиком, так и при нажатии мышкой на боковуй ползунок

{% notice w %} Скролл влево-вправо не допускаеться. {%end%}

Коментарии

Аватарки

Все картинки должны быть в одинаковом стиле, либо квадратные либо круглые. Имя автора коментария должно соответствовать полу того кто на фото. Таже нацианальность коментаторова на фото должны соответствовать гео сайта. Те если сайт на арабском языке - фото должны быть с арабами, если на европейские страны - на фото должны быть европейцы.

Если есть коментарии от врача, фото и имя врача должно совпадать с именем и фото вначале статьи (обычно он там есть)

{% slider dark %} https://i.imgur.com/cgcjTYT.png https://i.imgur.com/nuWUZ7Q.png {% end %} {% notice w%} Не должно быть дублирование аватаров, когда фото одно, а имена разные. {% end %}
Даты коментариев

Даты коментариев могуть быть как датой, так и временем, сколько прошло с момента его оставления. Коментарии могуть отображаться сначала новые, потом старые, так и наоборот. Главное чтобы они небыли в перемешку.

{% note w %} Пропажа коментариев Иногда происходит пропадание коментариев в мобильной версии сайта. Например в десктоп версии коментариев 10 - а в мобильной версии 7. Такого быть не должно. {% end %}
Город коментатора

Город коментатора должен соответствовать той стране, для которой предназначен сайт.

Оффер

Оффер должен соответствовать тому, что установлен в админке. По тексту не должно быть никаких других офферов, за исключением когда они упонимаються специально. Например когда действие продутка сравниваеться с другими.

{%img%} https://i.imgur.com/kYjHl7I.png {%end%}

Все картинки с продуктом по сайту и в коментах должны соотвецтвовать. В коментариях фото продукта должны быть "живыми".

Баг кэша

Тк сайты кэшируються, могут возникать подобные ошибки

{%img%} Выглядит это примерно так $ https://i.imgur.com/6SNnNu2.png {%end%}

Баг связан с ошибкой в постороении кеша. И дабы ее решить - нужно сбросить кеш на этом ленде.

{%code safe%} https://ваша_ссылка?PageSpeed=off {%end%}

PageSpeed - фича, предназначенная для ускорения открытия промо. Добавив ей значение off, можно сбросить кэш сайта.

Баг кеширования может быть разный, вплоть до слетания стилей. Поэтому перепроверяйте всегда с хвостом, чтобы убедиться, что это не кеш, а реальная бага
